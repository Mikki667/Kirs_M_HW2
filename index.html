Counter Documentation

Reusable Counter Component

Overview

This project demonstrates a reusable counter component built using JavaScript Object-Oriented Programming. The goal of the component is to manage its own state and update the interface automatically when the user interacts with it.

Each counter instance keeps track of its value internally and controls its own buttons and display. This makes it possible to create multiple counters on the same page without conflicts or shared state issues.

The component also prevents negative values and visually indicates when certain actions are not available. This helps create a more predictable and user-friendly interaction pattern.

How To Use

To use the counter component, first create a container element in HTML where the counter will be rendered. After that, instantiate the Counter or StepCounter class in JavaScript and pass the container selector as a parameter.

The Counter class accepts an optional initial value which defaults to 0. The StepCounter class extends this functionality by allowing a custom step value for incrementing and decrementing.

Each counter mounts itself into the provided container and handles its own UI updates through internal methods. This keeps the logic organized and separates state management from layout structure.

Key Features

State-driven UI updates

Multiple independent counter instances

Reset functionality with console feedback

Step-based counter using class inheritance

Conditional button states when value is zero

Modular styling structure built with SASS
